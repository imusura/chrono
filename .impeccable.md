## Design Context

### Users
- **Clients** — submit tickets and track their own issues. Non-technical, want clarity and simplicity. Should feel confident their issue is being handled.
- **Agents/Developers** — read and respond to client tickets, triage, assign, update status. Spend extended time in the app. Need efficiency without visual fatigue.
- **Admins** — full access, user management. Power users who need overview and control.

All users benefit from a lightweight, uncluttered interface that surfaces the right information without overwhelming.

### Brand Personality
**Professional, clean, energetic.**

- Professional: trustworthy, competent — clients should feel their issues are in good hands
- Clean: generous whitespace, restrained color, no visual noise
- Energetic: not dull or sterile — subtle color accents and purposeful use of badges/icons add life without clutter

### Aesthetic Direction
- **Visual tone:** Modern and airy — generous whitespace, clean layouts, breathing room between elements
- **Color strategy:** Neutral base (grayscale) with semantic color accents at low opacity (10% backgrounds with colored text/icons). Color communicates meaning (status, priority, type), not decoration
- **Icon treatment:** Squared containers with colored icon + same-color background at 10% opacity (validated pattern)
- **Badge treatment:** Rounded-md corners, colored text on matching low-opacity background, no heavy borders
- **Theme:** Light mode primary. Dark mode supported via CSS infrastructure
- **Anti-references:** Dense helpdesk UIs (Capacity-style) with competing panels, badge soup, and no focal point. Avoid: full-saturation color everywhere, cramped layouts, heavy borders separating everything

### Design Principles

1. **One focal point per view** — each screen should have a clear hero element. The ticket list OR the detail sheet commands attention, never both at equal weight.

2. **Color means something** — every color accent maps to a semantic concept (status, priority, type). Never use color purely for decoration. Neutral by default, color by exception.

3. **Whitespace over borders** — separate content with generous spacing, not heavy lines. When borders are needed, keep them subtle (`border-transparent` or low-opacity).

4. **Low density, high clarity** — fewer elements per row/card, each with clear purpose. Three readable items beat six cramped ones. Prioritize scannability for agents who triage all day.

5. **Consistent component language** — badges, icons, and status indicators follow the same visual pattern everywhere they appear. If a priority badge looks one way in the table, it looks the same in the detail sheet.
